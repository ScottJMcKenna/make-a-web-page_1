5 Ways to Think Like a Programmer

  1. Procedural Thinking

    Procedural thinking involves creating perfectly clear and unambiguous instructions for a computer to follow. It may be easy to tell a person how to sort a deck of cards, but getting a computer to do that requires more thought. But once you've learned to think this way you'll find that computers can do it much faster.

  2. Abstract Thinking

    Abstract thinking means finding similarity, or as programmers would say, generality amongst seemingly different things.In this Nanodegree you will visit hundreds of Udacity web pages. It would be impractical for a programmer to specifically program each of these pages individually. The Udacity programmers used the power of what's called abstraction to avoid unnecessary repetition of work.

  3. Systems Thinking

    Systems thinking happens when you break a big problem down into smaller pieces. Programmers do this when they create a plan (often on paper) for how a program will work. It involves big-picture thinking and decision-making about a problem and how different pieces of a program can work together to solve it. If this is unclear now, don't worry! You'll understand what this means much more clearly by the end of the Nanodegree.

  4. Technological Empathy

    "Technological empathy" comes in many forms. For example, computer empathy is the ability to understand what a computer is, how it works, and what it's good and bad at doing. A computer is a tool, as are the programs we use and the programming languages used to write them. It's almost impossible to program anything substantial without a basic understanding of how these tools work.

  5. Debugging

    Debugging is a systematic process of relentlessly identifying the cause of a computer program that doesn't work. When a program doesn't work, it's because there is a mistake (also known as a "bug") somewhere in the computer code. Since these mistakes are an unavoidable part of programming, every good programmer has a system for fixing them and that system usually looks something like this:

    1. Collect evidence (what makes this program fail?)
    2. Generate theories (what may have caused this problem?)
    3. Test those theories (if my theory is correct, how could I find out?)
    4. Fix the problem

Web Pages, Documents, and Structure

The Web

The Web is essentially a collection of HTML documents. Files hosted on The Web include: plain text, HTML, images, video, and music.

The Web works starting with you, and works through your computer/browser, to the internet, HTTP, the servers that host your requested infomation, and then returns on the same path back to you. A web page is a text document written in a language called HTML. Web browsers read these documents, and then interpret and display them.

HTML

HTML --HyperText Markup Language-- is the language of coding, and is made up of text content (what you see), markup (how it's arranged), references to other files (images or video), and links to other pages.

HTML Markup consists of tags. And Tags consist of an opening tag, the content that it effects, and then a closing tag. Some examples of tags include:

The B Tag --bold-- <b>CONTENT</b>

The Em Tag --emphasis-- <em>CONTENT</em>

It's often a good idea to make your closing tag as soon as you make your opening tag so that you don't cause any unexpected problems. Some text editors will automatically generate closing tags so that you don't forget.

Anchor Tags -- to create a link in HTML the code would begin with an Anchor Tag and be written like this:
<a href="www.chirpradio.org">CHIRP</a> and appear as this: CHIRP

Image Tags -- to add an image to an HTML document you would use an image tag structured like this:
<img src="url"alt="text"> and it would appear as this:

Image tags do not require a closing tag because they are --what is known as-- a Void Tag.

You may be wondering why anchor tags use the href attribute while img tags use the src attribute. And you may become frustrated when (not if) you make the mistake of using one when you really need the other. Because when you make this mistake, your HTML will not work.

Break Tags --The proper use of Whitespace will allow you to either display multiple lines of text or display them on a single line. For single lines of text you would use a <br> tag --or a Break Tag. A Break Tag is considered an Inline Element, the same as a B Tag, Em Tag, or a Image Tag.

P Tags --for text that will require some space, kind of like a text box, a P tag: <p> --or a Paragraph Tag-- would be appropriate. A Paragraph Tag is considered a Block Element since a paragraph will have both height and width.

HTML Attributes
An HTML Attribute is structured like this: <tag attr="value">CONTENT</tag>

Container Elements
Other Inline Elements, like Break Tags, are: Anchor Tags ("a"), Span (class="foo"), Strong Tags, and IMG Tags.
Other Block Elements, like a Paragraph Tag are: DIV Tags (class="bar"), and Form Tags.

Components of a HTML Document

  1. <!doctype html> - document type
  2. <html> - tag
  3. <head> - head - metadata - javascript, css included here
  4. <title>title!</title> - title
  5. </head>
  6. <body> - body
  7. <b>content</b>
  8. </body>
  9. </html> - closing tag

Lesson 2: Creating a Structured Document with HTML

Tools

HTML --this language is like a house, a structure. Text written in a document follinw its own rules & syntax as opposed to traditional grammar. A basic word in this language is called a Tag. The browser then turns this into a tree.

CSS --is the decoration and interior design.This defines how elements will look on the page using it's own syntax & rules - font, color, position, etc.

Java Script --is the appliance that makes things work.

DOM

HTML and CSS are both languages. HTML controls the structure of a web page, and CSS controls the style --how it looks. When Programmers talk about the "DOM", they are talking about the tree-like structure.

The "tree-like structure" comes from the fact that HTML elements can have other elements inside of them. You can draw this relationship like a family tree. My mother had multiple children. So did her mother, and so on... In a browser, this structure shows up as a series of nested boxes. There are boxes inside of boxes inside of boxes, and so on...

Boxes

All web pages can be broken down visiaully into groups of individual boxes. Looking at any web page, a programmer will begin by seeing the bigger boxes --header, body, footer, etc.-- and then will be able to view smaller content therein as individual boxes unto themselves. When you read an HTML document as text, you see a wave of changing indentations going up and down the left side of the document. The more indented an element is, the more deeply nested it's corresponding "box" is.

Lesson 3: Adding CSS Style to HTML Structure

Utilizing CSS to Avoid Repetition

CSS --Cascading Style Sheets-- allows programmers to avoid unecessary repetition by applying a style consistenly across all HTML elements, that they assign to a unique class. Since each page is made up of many elements, the programmer can give each one its own look according the design requirements of her/himself or of the needs of a client. By enabling a set of stylistic rules for every element within a given class, the need for the programmer to format each element individually is eliminated. This saves, time, cost, and streamlines the troubleshooting process for any obvious errors.
